# Mindwtr 1.1.0-rc.5

This is the fifth release candidate for Mindwtr 1.1.0. Use it for tester channels only. Prepare final stable `1.1.0` notes separately when the stable release is cut.

## RC Testing Focus

- Complete a recurring task that only has a due date (or only a review date): it must leave Focus and Next actions immediately and come back when its next date arrives — no more identical-looking next instance inviting repeated done-swipes (#843). Editing a repeat's schedule so it defers should also drop its Today's Focus star.
- Android: turn on the Quick capture notification, fully close the app (swipe away in recents), and confirm the notification returns as soon as anything wakes the app — opening it, or a task reminder firing — with task reminders off (#819).
- Sync a large library on Android and compare cycle time against rc.4: finishing a sync no longer re-reads the whole database, and reminder rescheduling skips scans that cannot change reminders (#766).
- Scroll and reorder inside a project on mobile: no settle-jump after lifting your finger, no top-jump when entering Select mode, and no blank list after leaving Task order mode (#831, #765, #784).
- On iOS, run the new "Add to Mindwtr Inbox" Shortcuts action from an unattended Automation with the app closed, then open the app and confirm the task landed in the Inbox (#845). Also verify "Open Mindwtr List" opens the chosen list instead of an error screen (#755).
- Attach files on desktop from odd locations (network mount, RAM disk): the app stores its own copy, failures are reported at attach time, and attachment rows show a paperclip only for files the app actually owns — links show a link icon with the path in the tooltip (#836).
- Self-hosted web: redeploy between visits and open a lazily loaded page like Settings > GTD — no more permanent "Importing a module script failed" (#844).
- Dropbox sync: wake the device from sleep and sync — a single dropped request must retry instead of claiming "No internet connection".
- Reopen the app within 25 minutes of leaving it: it should resume on the screen and project you left; after a longer break it opens fresh (#842).
- On the Focus screen grouped by tags, double-click a multi-tag task: exactly one editor should open and stay editable (desktop).

## Highlights Since rc.4

- Completing a due-only or review-only recurring task now defers the next instance until its date arrives, instead of leaving an identical-looking copy in Focus and Next actions.
- The Android Quick capture notification survives OEM app kills: any app activity — including a reminder firing in the background — re-pins it automatically.
- Mobile sync cycles are several seconds faster on large libraries, and reminder rescheduling skips unnecessary full-library scans.
- Project task lists on mobile scroll natively: no settle hiccups, no Select-mode top-jump, no blank list after reorder mode.
- New iOS "Add to Mindwtr Inbox" Shortcuts action captures tasks in unattended Automations without opening the app; iOS list-opening shortcuts work again.
- Desktop attachments are copied into app storage at attach time with honest owned-copy vs. pointer icons and a browse-to-link button.
- The self-hosted web app no longer permanently breaks after a redeploy due to service-worker cache poisoning.
- Dropbox sync retries transient failures with backoff instead of reporting a false "no internet" skip.
- Reopening the app within one Pomodoro block resumes where you left off.
- The Inbox Processor's Area dropdown matches the Project picker instead of rendering as a native OS select; the mobile Undo toast can no longer be suppressed by a desktop-only setting.
