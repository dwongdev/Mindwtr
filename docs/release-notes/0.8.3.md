# Mindwtr 0.8.3 (Draft)

> Draft release note for the next tagged release after v0.8.2. Update this file before tagging the actual release.

## Highlights

- Added DGT GTD import and internal Markdown links for tasks and projects, including typed navigation and deleted-item restore affordances.
- Improved day-to-day UX with AI clarify loading feedback, corrected mobile review-notification routing, and localized native date inputs on desktop.
- Continued the desktop/mobile refactor pass by splitting sync settings, task edit flows, review flows, inbox processing, agenda sections, and other high-churn UI modules into smaller units.
- Hardened sync behavior by preserving recoverable `createdAt` timestamps during merge repair, flushing pending writes before retry recovery, and keeping sync documentation aligned with the shipped delete-vs-live rules.

## Features & UX

- Added DGT GTD JSON/ZIP import support and repository docs for the import flow.
- Added internal Markdown links for tasks and projects.
- Added an inbox AI clarify loading state.
- Fixed mobile review notifications so they reopen the correct flow.
- Localized native desktop date inputs and mobile search placeholders.
- Kept incomplete Dutch translations out of the language pickers until the locale is ready.

## Sync, Reliability & Developer Ergonomics

- Preserved recoverable `createdAt` values when correcting invalid merge timestamps.
- Flushed pending writes before retry recovery in sync flows.
- Added MCP SQLite write-path coverage and bootstrap recovery logging.
- Narrowed desktop task-item subscriptions and continued sync/settings refactors to reduce blast radius in hot UI paths.

## Refactors

- Split mobile sync settings transport, backend panels, sections, and transport state handling.
- Split mobile quick capture, swipeable task items, project detail modals, task edit field renderers/controllers, review modal helpers/controllers, calendar view controller, and inbox processing controller.
- Extracted desktop inbox processing, agenda sections, list controls, selection state, project workspace, settings helpers, and task item container logic.
- Split core sync test scenarios and DGT import modules for easier maintenance.

## Docs & Notes

- Added Markdown links and DGT GTD import wiki docs.
- Aligned sync ADR/wiki language with the current ambiguity-window behavior.
- Expanded the keyboard, Pomodoro, reusable-list, and cloud-sync wiki guidance.
